# External Resources Catalog

Триггеры: "каталог скиллов", "где найти скилл", "MCP сервер", "найти MCP", "UI компоненты", "дизайн инспирация", "external resources", "источники", "найди готовый", "есть ли готовое"

## Контекст
Когда нужна внешняя ресурса: скилл, MCP-сервер, UI-компонент. НЕ писать с нуля — сначала проверить каталоги.

## Алгоритм поиска

### Шаг 1: Определи что нужно
| Нужно | Куда |
|-------|------|
| Скилл для AI CLI/агента | skills.sh → awesome-agent-skills → GitHub |
| MCP-сервер для интеграции | mcp.so → smithery.ai → npm/github |
| Готовый UI-компонент | 21st.dev → shadcn/ui → scrollxui.dev |
| AI-генерация UI с нуля | v0.dev |
| Анимации/эффекты | Aceternity UI → motion-primitives |
| Визуальная инспирация | Dribbble → Mobbin |

### Шаг 2: Поиск
1. Зайди в первый каталог из таблицы
2. Используй поиск/фильтр по ключевым словам
3. Если нашёл → скопируй/подключи → перейди к Шагу 3
4. Если не нашёл → следующий каталог
5. Если все каталоги пустые → гугли `<тип> <название> github/npm`

### Шаг 3: Проверка работоспособности
- Скилл: прочитай SKILL.md, проверь триггеры, структуру
- MCP: проверь конфиг, версию протокола, требования
- UI: проверь зависимости, совместимость с твоим стеком
- Если не работает или устарел → вернись к Шагу 2, следующий каталог

## Обработка тупиков

Если все каталоги не дали результата:
1. Загугли `<тип> <название> github` — может быть в личном репо
2. Попробуй альтернативное название (browser-automation ≠ web-scraper)
3. Проверь npm: `npm search <тип>`
4. Если совсем ничего — создай скилл сам, зафиксируй что готового не нашлось
