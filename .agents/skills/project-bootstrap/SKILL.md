# Project Bootstrap

Триггеры: "новая директория", "новый проект", "начни работу", "создай AGENTS.md", "project bootstrap", "инициализируй проект", "новая папка проект", "подготовь проект", "настрой правила для проекта", "init project rules"

## Контекст
ИИ начинает работу в новой директории (проект, папка, репозиторий). Нужно чтобы любой CLI/IDE загрузил правила из `~/AGENTS.md`.

## Алгоритм

### Шаг 1: Проверка
1. Есть ли `AGENTS.md`, `QWEN.md`, `CLAUDE.md`, `GEMINI.md`, `.cursorrules` в текущей папке?
2. Есть ли `~/AGENTS.md` (Linux/macOS) или `%USERPROFILE%\AGENTS.md` (Windows)?

### Шаг 2: Действие

**Если `~/AGENTS.md` НЕ существует:**
→ Создай минимальный `~/AGENTS.md` с правилами #1 (секреты), #2 (думай умно), #3 (✅😈), ссылкой на полный файл если появится.

**Если в проекте уже есть AGENTS.md:**
→ Не перезаписывай. Открой, прочитай. Если нет ссылки на `~/AGENTS.md` → добавь в начало: `# Главный файл правил: ~/AGENTS.md`. Если уже есть → ничего не делай.

**Если в проекте нет AGENTS.md:**
→ Создай все применимые файлы (см. таблицу ниже).

### Шаг 3: Создание файлов

| CLI/IDE | Файл | Содержимое |
|---------|------|-----------|
| Qwen Code | `<project>/.qwen/AGENTS.md` или `<project>/QWEN.md` | Ссылка на `~/AGENTS.md` |
| Claude Code | `<project>/CLAUDE.md` | Ссылка на `~/AGENTS.md` |
| Codex | `<project>/.codex/AGENTS.md` | Ссылка на `~/AGENTS.md` |
| Gemini CLI | `<project>/GEMINI.md` | Ссылка на `~/AGENTS.md` |
| Cursor | `<project>/.cursorrules` | Ссылка на `~/AGENTS.md` |
| Иные | `<project>/AGENTS.md` | Ссылка на `~/AGENTS.md` |

**Содержимое каждого файла:**
```markdown
# Project Rules

Следуй глобальным правилам: `~/AGENTS.md`
```

## Почему это работает

- Мини-файл ~10 слов (~75 токенов) — дёшево, всегда в контексте
- Кроссплатформенно: `~` для Linux/macOS, `%USERPROFILE%` для Windows
- Кросс-CLI: создаёшь все форматы — любой инструмент подхватит правила
- Ссылка, не копия — при обновлении `~/AGENTS.md` все проекты получают актуальные правила

## Примеры

✅ Правильно:
- Создал QWEN.md + CLAUDE.md + .cursorrules → все 3 CLI загрузили правила
- В проекте уже был CLAUDE.md → добавил ссылку на `~/AGENTS.md` в начало
- `~/AGENTS.md` не было → создал минимальный с 3 критическими правилами

❌ Неправильно:
- Скопировал весь `~/AGENTS.md` в проект (дубли, рассинхрон при обновлении)
- Создал только QWEN.md, Claude Code не загрузил правила
- Перезаписал существующий AGENTS.md проекта (потерял проектные правила)
